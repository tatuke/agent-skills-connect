为什么要 connect？

如果我现在有一个能自动识别上传内容、保存并自动总结的技能 A，还有一个能自动搜集热点新闻的技能 B。若我想在搜集热点的同时提取技术内容并保存，直接创建一个能保存热点的新技能 C 看起来顺理成章。但是，如果用某种方法将它们组合起来会怎样呢？理由如下：

1) A 和 B 组合后，我们仍可保留这两个原型，从而启发后续的想法。
2) 更基础的技能（比如 A）可以与其他新技能结合，实现更丰富、本地化的功能。
3) 假如 A 和 B 来自社区且正在持续改进中（例如 A 支持了更多数据库，或采用了更合理的数据库范式），那么作为它们“新生儿”的 C，如果仅停留在本地，就会停滞不前（它的父母在进化，而它却没有）。
